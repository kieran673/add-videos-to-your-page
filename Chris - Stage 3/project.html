Introduction to HTML

Elements

An element is made up a of start tag, the content and the end tag.They are written in the following format:

<tag> content </tag>

Tags

A tag is a code that describes how the contents must be formatted. Almost all tags have an opening and closing tag, EXCEPT void tags. A tag is created by putting the name of the tag in angled bracket, an end tag is the same but with a slash after the opening angled bracket and before the tag.
Tags can also be INLINE (the content is displayed without a new line break) or BLOCK (The content is displayed within its own box)
Below is a table giving details of just a few tags,

Name Tag Description Inline/Block Void?
Bold <b> Bolds the contents Inline ✘
Emphasis <em> Italicizes the contents Inline ✘
Link <a> Changes the contents into a link Inline ✘
Image <img> Adds an image Inline (However img acts
more like an "inline block")		 ✘
Break <br> Adds a line break to the sentence Inline ✔
Paragraph <p> Creates a new paragraph Block ✘
Span <span> Groups the contents together Inline ✘
Division <div> Groups the contents together in a block Block ✘

Attributes

Attrubributes are put inside the opening tag to modify them.
Here are just two examples:

<img src = "smily_face.jpg" alt = "a smily face" height = "42" width = "42">
  • src: Gives the source of the image. This tells the program where to get the picture
  • alt: If the picture is unable to load this tells the program what text to show instead.
  • height: Specifies the height of the image in pixels
  • width: Specifies the width of the image in pixels

<div class = "description example_class ThisIsAClass" id = "example">
  • class: gives the element a class which interests with several other things (such as a the CSS file) a div can have several classes seperated by a space.
  • id: Similar to class, however a element can only have ONE id.

There is an incredible ammount of tags and attributes. A good place to see them all is this site: http://www.w3schools.com/tags/default.asp

Structured documents

HTML is written in a tree like structure. What this means is that each HTML element can have a "parent" and a "child". When appearing in a browser these appear as nested boxes, we have boxes appearing in boxes appearing in boxes. This concept is especially important when dealing with styles as "children" will inherit their "parents" styles but any styles in the "child" with take priority.

What is CSS

Cascading Style Sheets (CSS) is code written to control the "style" of HTML elements.
It is written in the following format: Selector {property: value;}
The selector defines what parts of the html to effect. The table below lists some of the more common selectors,

Selector Example Example description
Element p selects all <tp> elments
class .info selects all elements with the class "info"
#id #first_title selects the element with the id "first_title"
Everything inside the curley brackets is a "rule" that will be applied to every instance of the selector. .At the end of our rule we add a ";" to tell the program that we are finsihed.
First is the declaration. For example background-color (which would control the background color).We end the declaration with a colon
After the declaration is the value (for example: black).

so if we wanted to effect every h1 text and turn it blue with a red background we would write:

How to Include CSS in a web page

There are three methods to include CSS in a web page

Method 1: Write your style inline with your HTML



Write the CSS inside the individual HTML element. This is not recommended because it requires a lot of repeated code but is useful if you want a certain section to have it's own style.

Method 2: Write CSS in the of the HTML



Inside the head element of the HTML add a style element. Write the CSS inside that style element.

Method 3: Link your HTML to a separate CSS file



Put all of your CSS code in a file separate from the HTML file. Add a tag inside the element of the TML file(Since is a "void tag" you don't need to add a closing) then add the following attributes to the : * rel="stylesheet" * type="text/css" *href="main.css"

Cascade

What happens if the HTML finds two styles rule that apply to the same section? This is where the order is important. Styles are applied in the following order with definitions further down the list overwriting previous definitions
  • the default style of a browser (different browsers have slightly different styles)
  • stylesheet in a separate file (this is what you will be mostly using)
  • stylesheet inside HTML (this can be done for small projects but is not ideal)
  • inline style in an element (this can also be done but should be avoided)

What is the difference betwen 2 + 2 and "2" +"2"?


In python 2 is a number and "2" is a string. This means python treats them differently. For example when using the + operator with numbers would add the numbers together but using them with strings would concatenate the strings. So 2 + 2 would give 4 whereas "2" + "2" would give "22".

Variables

A variable is a symbol representing a value that can be reused. The value can assigned with the "=" sign, for example My_variable = 2.
Variables give us a way to improve the readability of the code and reuse a value over and over again. We can also use them to record and follow changes. For example we can add 2 to My_variable with My_variable += 2. Now My_variable has the value of 4.

Lists

A data structure is a sequence of data. The most basic type of data structure is a list.
Lists are sequence of elements surrounded by square brackets for example ["one", "two", "three"]. Each element is assigned a number which is its position or "index". The first index is zero, the second is one, the third is two and so on.
This indexing allows us to call specific indexes from the list. In the example above we could use [1] to call "two" or [1:3] to call "two", "three".
What makes lists so useful is their mutability, The ability to change the list. A common method is append. Written as list.append(x) (where list is the list we are mutating and x is the value to be added) this allows us to add a new piece of data to a list.
Append is very similar to the operate +, however append will mutate a list but + will create a new list.

if/elif/else

We use if, elif and else to test if the conditions after the if and elif are true before executing the code. If the conditions are not met then that part of the code is skipped.
We use else if we want code that will execute if none of the conditions in if and elif are met. We use "if" first, followed by any number of "elif" and end with else. The order is important as once python finds a condition that is True it stops checking the other "if".

for:

These loops iterate over a given sequence.

while:

These loops repeat as long as a certain condition is met.

Function

A function is a piece of code, then when called (by using the name of the function) executes that code. A function must be made and then used. Functions are made by starting a line of code with the keyword def (which stands for define) and then giving a function name followed by the function parameters in parentheses. when used (or called) we write the name of the function but place actual values into the parentheses.
Functions are extremely useful for a programmer since they no longer need to keep rewriting a piece of code that they use over and over again.
A very important part of the function is the return statement, this is what produces the output, without this statement the function will run through the code but then will not actually give python anything to use.

Object Orientated Programming

Abstraction

One of the most powerful tools we have in programming is abstraction. Abstraction is the ability to give a name to a thing, so that it represent the whole process or program.
Why is this useful? Let's imagine you've just moved to a new country and you mother's coming to visit but she can't read any of the signs. When you tell her the way to your house you have to describe every street. OK, you have go about 80 meters down the road until you reach a cross road then take a left untill you see the temple then take the next right then stay on the left lane when the road splits then… You quickly grow frustrated and ask your friend Steve to pick her up. Unlike your mother he's lived there for a few years so all you have to tell him is to head down Sukhumvit 101, turn into Soi 35 then when you reach the 7-11, turn right.
Instead of telling your mother all the directions to reach the 7-11 in Soi 35 we just tell Steve Sukhumvit 101. In Steve's head there is map of Bangkok and in that map is Sukhumvit 101, unlike your mother who has be told every single direction.
Steve has an abstraction of the Bangkok streets and with that we can give him directions to our home significantly faster than our mother. Imagine if no-one had these abstractions of maps in their heads, imagine trying to go just one week where whenever you were trying to meet up with someone you had explain every little detail to get there. Now imagine you also had to explain what a street or a building was or what left and right were.
Abstraction allow us to easily repeat and reuse an idea, which makes writing and reading our programs much easier. We have already used abstraction many times already, functions are the most obvious kind of abstraction. We create our function (the directions to my house for example) and then we can at any time easily call the function. Instead of writing out lines and lines of code we just need one (go_to(Sukhumvit 101 Soi 35)).

Objects

Programming languages like python use what is called object orientated programming. This type of programming creates programs that are designed from objects that interact with each other.
Most programming languages are class-based. We create the class first and then create an instance of that class. This instance is our object.
If we look at our example in abstractions we might have a class called "road", which might contain data like name, directions to the road, and maybe other nearby places. So then we would create an instance of this class and call it "Sukhumvit" we'd then add the directions to this road and nearby places (Which would be objects themselves, such as the object "7-11" or an instance of the class "Friend_house")

Definitions

Class
A class is like a template or a blueprint. We use classes to produce objects. When making a class the name of the class must have a capital letter. inside the class call a function called __init__ , which initializes or creates space in memory for a new instance. The first information placed into init is ALWAYS Self.
Object
An object or an instance is a one time appearance of a class. For example in real life we have our class of dog. All dogs have things in common but we can produce an object of a dalmatian or a poodle.
Method
A method is a function that is defined inside a class.
Module
A module is a collection of python definitions and statements collected into one file. These are usually definitions with a similar theme that would be commonly used together such as algebra or drawing a shape. We use these with import X for example import turtle. This will import the module allowing you to use any definitions inside.
Library
A library is a collection of modules. Python comes with The Python Standard Library, a large collection of commonly used modules. We can see this library at this website.
Class Variable
A class variable is a variable defined at the class level. These variables will remain the same across all instances of a class.
Inheritance
Inheritance is the concept that a Child can gain certain traits from its Parent. This is very useful in programming as it allows us to reuse code.
Method Overriding
We inherit methosds from a parent class as well. Inside the Child class we can override the method so that the child uses the new method instead of its Ancestor's method. This is extremely useful in programming as it allows us to reuse code but still allows us to modify or tweak the code to suit the specific purpose of the Child.
